## 2026-04-25 - CSV Injection via Shareable Shortlists
**Vulnerability:** A CSV Formula Injection vulnerability was found in the shortlist export functionality (`ShortlistDrawer.tsx`).
**Learning:** The application allows importing shortlists via base64 encoded URL parameters. This means a malicious user could craft a URL containing a shortlist with notes that execute formulas (e.g., `=cmd|' /C calc'!A0`). If a victim opens this link and exports their shortlist to CSV, spreadsheet software like Excel could execute the malicious formula. This is a surprising risk vector where client-side only state (localStorage) becomes vulnerable through URL sharing capabilities.
**Prevention:** Always sanitize user-provided text fields (like `notes`) when exporting to CSV, especially when the data can originate from external sources like shareable URLs. Prepend a single quote (`'`) to strings starting with formula trigger characters (`=`, `+`, `-`, `@`, `\t`, `\r`) to force spreadsheet applications to treat them as plain text.
## 2026-05-01 - Reverse Tabnabbing Vulnerability
**Vulnerability:** A Reverse Tabnabbing vulnerability was found when opening Google Maps directions and OneMap attribution links.
**Learning:** `window.open` without `noopener` or `noreferrer` allows the newly opened tab to access the original window's `window.opener` object, which can be manipulated to redirect the user to a malicious page. This is a classic vulnerability when linking to external resources using `target="_blank"`. It was missing in explicit programmatic calls (e.g., `window.open`) and standard `a` tags.
**Prevention:** Always use `"noopener,noreferrer"` as the third argument to `window.open` and `rel="noopener noreferrer"` in standard HTML link elements when opening new tabs (`target="_blank"`), especially for untrusted/external origins.
## 2026-05-09 - Missing HTTP Security Headers on Cloudflare Pages Deployment
**Vulnerability:** The app had no HTTP security headers, leaving users exposed to MIME-type sniffing, clickjacking, and referrer leakage. Notably, the app encodes user filter state (town, flat type, budget range, etc.) and coordinates into URL query parameters — these would be sent as the full Referer header to external origins like OneMap (map tiles) and Google Maps (directions), leaking browsing context.
**Learning:** Static SPAs deployed on Cloudflare Pages require an explicit `public/_headers` file to set HTTP security headers. Vite copies `public/` as-is to `dist/`, so placing `_headers` there ensures it reaches the deployment. The `Referrer-Policy` header is easy to overlook but particularly impactful here because user filter state lives in URL params.
**Prevention:** For any Cloudflare Pages SPA, add `public/_headers` early in the project. Use `Referrer-Policy: strict-origin-when-cross-origin` as the default. Be careful with `Permissions-Policy` — audit which browser APIs the app actually uses (this app uses `navigator.geolocation`) before restricting them.
## 2026-05-24 - Multiline CSV Formula Injection
**Vulnerability:** A CSV Formula Injection vulnerability was found in the shortlist export functionality, specifically targeting multi-line text input (`ShortlistDrawer.tsx`). While basic formulas were being sanitized via replacing `^\s*[=+\-@\t\r|]`, the multiline modifier (`m`) and global modifier (`g`) were absent from the regex.
**Learning:** Spreadsheet software processes formulas on a per-cell basis, and in typical scenarios, only the very first character of the entire string evaluated determines if it's a formula. However, if a cell is interpreted as multi-line string by certain parsers or depending on export formatting, the formula injection character must be sanitized strictly at the start of any multiline payload line. Without the `gm` flags in the RegExp replacing function, line breaks within notes allowed attackers to place malicious formula commands on lines 2+ bypassing the initial guard.
**Prevention:** Ensure any sanitization regex guarding string formatting, especially CSV injection, applies to the beginning of every single line using `gm` flags, e.g. `/^[ \t]*[=+\-@\t\r|]/gm`.
