# Repository Structure

## Directory Layout
- **`.kiro/`**: Centralized workspace steering and configuration.
  - **`steering/`**: Persistent repository rules, architecture, and UI standards.
  - **`specs/`**: Active or retained feature and bugfix specifications.
  - **`settings/mcp.json`**: Workspace MCP configuration for repo-relevant tool integrations.
- **`.agents/`**: Local agent tooling only. Keep it untracked and out of repository history.
- **Generic external tooling**: Do not vendor generic shadcn or design-system agent bundles into this repo. Use workspace MCP config or external installs instead.
- **`src/`**: React application source code.
  - **`components/`**: UI components.
  - **`components/ui/`**: Base Shadcn components.
  - **`hooks/`**: Custom React hooks.
  - **`lib/`**: Domain logic, formatters, and i18n helpers.
  - **`types/`**: Zod schemas and TypeScript interfaces.
- **`scripts/`**: Build-time data pipeline.
  - **`lib/`**: Pipeline processing modules and shared sync helpers.
- **`public/data/`**: Precomputed data artifacts (JSON/GeoJSON). Generated by `scripts/sync-data.ts` and deployed directly to Cloudflare Pages. **Not tracked by git.**
- **`data/cache/`**: Intermediate build caches (e.g., geocodes). Preserved across CI runs via GitHub Actions cache. **Not tracked by git.**
- **`tests/fixtures/public-data/`**: Minimal static data fixtures for unit and E2E tests. **Tracked by git.**
- **`tests/`**: Vitest and Playwright test suites.

## Naming Conventions
- **Components**: `PascalCase.tsx`.
- **Hooks**: `useCamelCase.ts`.
- **Lib/Utils**: `camelCase.ts`.
- **Data Artifacts**: `kebab-case.json`.
